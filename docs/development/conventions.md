# Code & Naming Conventions

*Canonical reference for naming and code-style conventions across AĒR's three languages, plus the project's domain-term canon. Authored in **Phase 140 (Naming & Code-Convention Unification)** as the reference the rename pass checks against, and kept as the standing convention doc. The mechanically-enforceable subset is locked by linters (see §6); the rest is reviewer guidance.*

> **The load-bearing rule (read first).** A name that crosses a **boundary** is a *contract*, not a free rename: OpenAPI field/operationId, ClickHouse column, Postgres column/enum value, URL-grammar key, NATS subject, content-catalog `entityType`/register key, the machine `probeId`. These change only via an explicit migration + codegen, never as part of a style pass. Everything else (local variables, private functions, internal type *names*, component/file names, comments, user-facing copy) is freely renameable to match this doc. **Renaming a Go struct field or TS type does not change the wire contract** — the JSON/`mapstructure` tag (Go) or the compact URL key (TS) is the contract; keep the tag/key, rename the identifier.

---

## 1. Domain-term canon (one spelling per concept)

The conceptual vocabulary is fixed by CLAUDE.md + the ADRs. Use exactly these spellings in code, types, comments, and UI copy. Retired terms in the right column must not appear in new code (and are purged from old code in Phase 140).

| Canonical | Meaning | Retired / wrong spellings (do not use) |
|---|---|---|
| **probe** | a corpus unit (`probe-<n>-<iso2>-<corpus-class>`) | — |
| **source** | one publisher within a probe | — |
| **pillar** (Aleph / Episteme / Rhizome) | analytical lens, determined by presentation | `viewingMode` as a *type name* (the URL key `viewingMode`/`activePillar` is contract) |
| **presentation** | a cell's visual form (`time_series`, `distribution`, `cooccurrence_network`, …) | `viewMode` / `ViewMode` as a *type name*; "view-mode matrix" |
| **panel** | one analytical unit (presentation × scope × window) | — |
| **cell** | the rendered presentation inside a panel | — |
| **scope** / **ScopeGroup** | the source-set a panel queries | `scopeGroup` (lower-camel) for the type |
| **window** | a panel's time range | — |
| **Workbench** | the analytical surface (Surface II) | **"Function Lane(s)"**, "Surface II" (as a user-facing name), `lane` |
| **Atmosphäre** / **Reflexion** | the other two surfaces | "four surfaces" (there are **three**) |
| **Dossier** | the global probe search/catalogue overlay | — |
| **discourse function** (`epistemic_authority` / `power_legitimation` / `cohesion_identity` / `subversion_friction`) | a source's primary register | — |

The presentation **values** (`time_series`, …) and discourse-function **keys** (`epistemic_authority`, …) are contracts (content-catalog + DB). Only the *type names* that hold them are renameable.

## 2. Go (`pkg/`, `services/ingestion-api`, `services/bff-api`)

Follow Effective Go + the standard library. Enforced subset in §6.

- **Initialisms are all-caps** (`ID`, `URL`, `API`, `HTTP`, `JSON`, `SQL`, `DB`): `ProbeID` not `ProbeId`, `DocumentationURL` not `DocumentationUrl`, `DBURL` not `DBUrl`. This is the ST1003 rule and the main Go drift Phase 140 fixes. **Exception:** identifiers generated by oapi-codegen (`generated.go`) and the handler methods that implement the generated `StrictServerInterface` (e.g. `GetSourceById`, whose name comes from the OpenAPI `operationId`) — these are contract-driven and excluded from the lint (see `.golangci.yml`). To change them, change the `operationId` in `openapi.yaml` and regenerate.
- **Exported identifiers** carry doc comments starting with the identifier name (`// Server handles …`). See `docs/operations/quality_register.md` §4 for the doc-coverage backlog (Phase 143).
- **Receivers**: short, consistent per type (`s *Server`, `p *PostgresDB`). One letter or a short abbreviation; never `this`/`self`.
- **Errors**: sentinel values are `Err*` (`ErrSourceNotFound`); wrap with `fmt.Errorf("…: %w", err)`.
- **Packages**: short, lower-case, no underscores, no plurals (`storage`, not `storages`).
- **JSON/mapstructure tags are the wire contract** — when you rename a field to fix casing, keep the tag (`ProbeID string \`json:"probeId"\``).
- **Formatting** is `gofmt` (enforced; never hand-format).

## 3. Python (`services/analysis-worker`, `crawlers/web-crawler`)

PEP 8 via `ruff` (already clean — 0 naming violations at Phase 140).

- `snake_case` functions/variables/modules; `PascalCase` classes; `UPPER_SNAKE` module-level constants; `_leading_underscore` for private.
- Public functions/classes carry docstrings where they carry meaning (intent/invariants/units) — not signature restatements (Phase 143 backlog).
- Line length 120 (`ruff` `line-length`).
- The `pep8-naming` (`N`) ruleset is enabled as a zero-cost ratchet (§6).

## 4. TypeScript / Svelte (`services/dashboard/src`, `packages/engine-3d`)

Already disciplined (0 filename/casing drift at Phase 140). The drift Phase 140 fixes is **domain-term**, not casing.

- **Files**: Svelte components `PascalCase.svelte`; `.ts` modules `kebab-case.ts`; directories `kebab-case`.
- **Identifiers**: `camelCase` values/functions; `PascalCase` types/interfaces/components; `UPPER_SNAKE` for true constants/magic numbers.
- **Type vs interface**: either is fine; be consistent within a module (`type` for unions/aliases, `interface` for object shapes is the prevailing split).
- **Imports**: use the `$lib/…` alias for anything under `src/lib`; relative `../` only within the same leaf directory.
- **Domain types** use the canon (§1): the presentation type is `Presentation` (not `ViewMode`); the pillar-id type is `PillarId` (not `ViewingMode`). The **URL keys** these serialize to (`v`/`view`, `viewingMode`/`activePillar`, the compact Panel keys `bn`/`ch`/`sb`/`fs`/`dl`) are contract — unchanged.
- **User-facing copy** uses the surface canon: **Workbench**, **Atmosphäre**, **Reflexion** — never "Function Lane(s)" or "Surface II".
- **Formatting** is Prettier (enforced); lint is ESLint + `svelte-check`.

## 5. Cross-boundary identifiers — deferred-with-reason (never renamed by a style pass)

These appear with non-canonical casing/spelling but are **contracts**. Listed so the rename pass skips them and reviewers know why:

- **OpenAPI**: field names (`probeId`, `sourceId`, `scopeId`, `comparedTo`, `viewerLanguage`), `operationId`s (drive `generated.go` method names like `GetSourceById`), and everything in `generated.go` / `src/lib/api/types.ts` (codegen output — Hard Rule #4).
- **URL-grammar keys**: `selectedProbes`, `dossier`, `activePillar`, `viewingMode`, `view`/`v`, the compact Panel keys `bn`/`ch`/`sb`/`fs`/`dl`, `aleph`/`episteme`/`rhizome`. (The legacy `?viewMode=` key survives only in the retired `/lanes/` 308-redirect, translating old bookmarks.)
- **Content-catalog keys**: `entityType` values (`view_mode`, `source`, `discourse_function`, `open_research_question`) and register keys (`semantic`, `methodological`).
- **DB**: ClickHouse `aer_gold.*` table/column names; Postgres column names + the `primary_function` enum values.
- **The machine `probeId`** (`probe-<n>-<iso2>-<corpus-class>`) — the stable load-bearing identifier everywhere.

## 6. Enforced (ratchet) subset

The mechanically-checkable conventions are locked in CI so drift cannot return:

| Language | Rule | Enforced by |
|---|---|---|
| Go | initialism casing (`ID`/`URL`/…) | `stylecheck` `ST1003` (`.golangci.yml`), run by `make lint` |
| Go | formatting, vet, unused, ineffassign | golangci-lint default set (`make lint`) |
| Python | PEP 8 naming | `ruff` `N` ruleset (`make lint`) |
| Python | unused imports/vars, format | `ruff` default `F` + `ruff format` (`make lint`) |
| TS/Svelte | retired-vocabulary ban (`Function Lane`, `ViewMode`, `ViewingMode`, `Surface II`, `WorkbenchScopeBar`) | ESLint `no-restricted-syntax` (`make fe-lint`) |
| TS/Svelte | dead code, format, types | `eslint` + `prettier` + `svelte-check` + `knip` (`make fe-lint`) |

Everything not in this table is reviewer judgment against §§1–4 — not a build gate.
