# AĒR — Public Methodology Portal

This is the **public** documentation for AĒR (ancient Greek ἀήρ, "air") — an
atmospheric sensor for human discourse. It contains what an external researcher
needs to **interpret the data the system makes visible**: the scientific
Working Papers, the per-probe descriptions, and the project's ethical
commitment.

!!! info "What you will (and won't) find here"
    This portal is deliberately scoped to **methodology and interpretation**.
    The engineering architecture (Arc42), the Architecture Decision Records, the
    operations runbooks, and the security model live in a separate **internal**
    documentation build that is access-controlled (ADR-040). They are not needed
    to read AĒR's published metrics, and exposing them to every reader would be
    inappropriate once the instrument serves external researchers.

## Where to start

- **[Manifesto](arc42/00_manifesto.md)** — the epistemological and ethical
  foundation. AĒR is a phenomenological, anti-surveillance instrument; the
  Manifesto states the non-negotiable commitments (collective anonymity,
  methodological transparency, no digital twins).
- **[Methodology (EN)](methodology/en/WP-001-en-toward_a_culturally_agnostic_probe_catalog-a_functional_taxonomy_for_global_discourse_observation.md)**
  — Working Papers WP-001 … WP-006: the functional probe taxonomy, metric
  validity, platform bias, cross-cultural comparability, temporal granularity,
  and the observer effect.
- **Probes** — per-probe descriptions: what each probe observes, how its
  sources are classified, and the documented limits of what it can and cannot
  measure.

## How to read AĒR's metrics responsibly

Every metric AĒR ships is **provisional engineering** unless a Working Paper
explicitly promotes it. Lexicons, models, thresholds, and parameters are
defaults, not validated scientific instruments. The Working Papers are the
authority on what a number means — and on what it does **not** mean.
